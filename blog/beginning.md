# What is Bedrock?

Bedrock is my attempt to create a deep-learning framework in c++ that is designed to improve the speed of training and inference of neural networks, relative to modern python libraries.

# Why create Bedrock?

I am (trying) to create a deep-learning framework from scratch because I believe it will allow me to gain a much deeper understanding of the mathematics and computer science behind the tools I use on a daily basis. Not only will I challenge my comprehension about ideas like computational graphs, back-propagation, and tensor manipulation, but I will also develop the skills of pursuing a long-term project, enforcing critical thinking, modular design, and code testing. I find this style of hands-on learning to be ten times more engaging, difficult, and rewarding. By the end of this project I will hopefully have achieved something that I am genuinely proud of, or in the best case scenario, something that someone else will find useful.

# What are my Next Steps?

From here, I need to identify the core stucture of the repository, and make several design choices that will determine how the development of the project will continue. My first design choice will be to create a blog, not only to demonstrate my thinking to a potential audience, but so that I can refine my thoughts and ideas before acting on them in code. Secondly, I think that creating this project in a low-level language like C++ is favourable as it will allow me to make optimizations in regards to memory management that would not be available in a high-level language like Python. This decision is something I will have to be incredibly careful with as the lack of guardrails may also lead to serious headaches and debugging down the road. From this point I will start with the fundamentals and create a tensor object which can be modified in ways that are essential to any deep-learning framework. During all of the stages throughout this project, I will use libraries like PyTorch and TensorFlow as inspiration to towards the development of my own code.
